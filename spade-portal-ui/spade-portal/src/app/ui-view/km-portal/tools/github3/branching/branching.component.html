GitHub3

Branching & Structure

  • Login to GitHub3 once you get the account.
  • There will be two kinds of Organizations in Github3.
    • AS-Community – Internal AS repository (Note:- when you request for a new Github3 account select organization as "AS-Community" for internal AS organization unless customer specific).
    • Customer Specific (e.g., Comcast) – Customer specific Repository.
  • Both will follow the below hierarchy of directories and branching mechanism.
  • Once Repository is created default Master branch will be created.
  • Under Master Branch create following Category directory structure before creating additional branches. Create more categories based on the need.
  • Create Branches from Master Branch based on need like Dev, QA, Pre-prod etc.
