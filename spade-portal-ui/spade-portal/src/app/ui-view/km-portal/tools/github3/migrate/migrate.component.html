Migrating to GitHub3

  • Review you existing repositories in Gitlab or other existing repositories.
  • Verify if existing repositories are customer specific or AS internal.
  • If it is AS internal, request GitHub3 account and specify AS-Community as organization.
  • If it is customer specific request GitHub3 account with out any organization and later create organization with customer specific name.
  • Once you have selected right organization create repository with domain or technology specific area (e.g.,NSO,ACI etc).
  • After creating the repository create categorization structure as explained earlier.
  • Migrate your existing Apps/Scripts from Gitlab to GitHub3, alternatively raise a support request with IT by providing following details:
    • Source repository name, Source Directory/File name, Destination Organisation/Respository/Category in the request for IT team to complete the migration.
